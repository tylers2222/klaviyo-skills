# Rewards Program

Replace this file with your business's actual loyalty/rewards program details. The skill consults this file when building loyalty, rewards, or membership components to use real program data instead of placeholders.

## Example Format

```markdown
# Rewards Rules

1. For every [X] points accumulated, get a $[Y] gift voucher
2. [X]% off [product category] at all times
3. Free shipping on all orders over $[X]
4. [Tier name] members get early access to sales
5. Birthday month: double points on all purchases
```

## What to Include

- Point earning rates (e.g., $1 spent = 1 point)
- Reward thresholds (e.g., 200 points = $5 voucher)
- Member-only discounts and which categories they apply to
- Shipping perks and minimum spend requirements
- Tier names and what each tier unlocks
- Special events (birthday rewards, anniversary bonuses, double points periods)
- Any exclusions or conditions

The more specific the data, the more accurate the components. Vague benefits like "exclusive offers" produce vague designs. Concrete numbers like "$5 voucher per 200 points" produce components that sell.
